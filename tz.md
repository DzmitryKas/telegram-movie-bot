# Техническое задание: Telegram-бот «Фильм на вечер»

## 1. Общее описание

Telegram-бот для быстрого подбора фильмов на вечер. Пользователь указывает предпочтения (жанр, настроение, длительность), бот предлагает релевантные варианты с рейтингами и информацией о том, где посмотреть.

---

## 2. Технологический стек

## 4. База данных

| Поле | Тип | Описание |

| id | serial (PK) | Уникальный ID |

| Поле | Тип | Описание |
|------|-----|----------|
| id | serial (PK) | Уникальный ID |

| action | varchar | 'like' / 'dislike' |
| created_at | timestamp | Дата действия |

### 4.3. Таблица `watched_movies`

| Поле | Тип | Описание |
|------|-----|----------|
| id | serial (PK) | Уникальный ID |

## 5. Функциональные требования

### 5.1. MVP (Phase 1)

#### Команда `/start`
- Приветственное сообщение
- Краткое описание возможностей
- Кнопка «Найти фильм»

#### Команда `/movie`
- Запуск сценария подбора:
  1. **Шаг 1**: Выбор жанра (inline-кнопки)
     - Комедия, Драма, Триллер, Ужасы, Фантастика, Боевик, Мультфильм, Документальный
  2. **Шаг 2**: Выбор настроения
     - 😊 Лёгкий, 🤔 Серьёзный, 🔥 Захватывающий, 😢 Трогательный
  3. **Шаг 3**: Длительность
     - ⏱ До 90 мин, ⏱ 90-120 мин, ⏱ 120+ мин, ⏱ Не важно
  4. **Результат**: Бот выдаёт карточку фильма:
     - Название (год)
     - Рейтинг IMDb / Кинопоиск
     - Краткое описание (2-3 предложения)
     - Постер
     - Кнопки: «👍 Нравится», «👎 Не то», «🎲 Ещё вариант»

#### Команда `/help`
- Список доступных команд
- Как пользоваться ботом

#### Обработка действий
- **👍 Нравится**: сохраняет в preferences, предлагает ещё
- **👎 Не то**: сохраняет в preferences, предлагает ещё
- **🎲 Ещё вариант**: выдаёт новый фильм с теми же фильтрами

### 5.2. Фаза 2 (Расширение)

#### Команда `/profile`
- Статистика: сколько фильмов просмотрено, любимые жанры
- История лайкнутых фильмов

#### Команда `/random`
- Мгновенная выдача случайного фильма без фильтров

#### Команда `/top`
- Топ-10 фильмов за неделю (по рейтингу TMDB)

#### Умные рекомендации
- На основе истории лайков/дизлайков
- Персонализированная выдача

### 5.3. Фаза 3 (Плюшки)

#### Команда `/couple`
- Режим «Парный вечер»: оба свайпают, бот находит совпадение

#### Интеграция со стримингами
- Показывает, где доступен фильм (JustWatch API)

#### Команда `/series`
- Подбор коротких сериалов (мини-сериалы, 1 сезон)

## 6. Внешние API

### 6.1. TMDB API

| Эндпоинт | Назначение |
|----------|------------|
| `/discover/movie` | Поиск по фильтрам (жанр, длительность) |
| `/movie/{id}` | Детали фильма |
| `/movie/{id}/credits` | Актёры и режиссёры |
| `/movie/{id}/images` | Постеры и фоны |
| `/genre/movie/list` | Список жанров |
| `/search/movie` | Поиск по названию |

**Авторизация**: Bearer Token (API Key)
**Лимиты**: ~40 запросов/сек (бесплатный тариф)

## 7. Нефункциональные требования

### 7.1. Производительность
- Время ответа бота: < 3 секунд
- Кэширование популярных запросов (Redis, опционально)

### 7.2. Надёжность
- Graceful shutdown
- Retry-логика для внешних API
- Обработка ошибок TMDB (фильм не найден → предложить другой)

### 7.3. Безопасность
- Хранение API-ключей в `.env`
- `.env` в `.gitignore`
- Валидация всех входящих данных (Zod)

### 7.4. Логирование
- Логирование ошибок в файл
- Логирование действий пользователей (для аналитики)
